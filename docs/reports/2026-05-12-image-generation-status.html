O que estava acontecendo com imagens

Texto via Gemini CLI funcionava porque o modo CLI local entra apenas no caminho de texto. Imagem não usa Gemini/Codex/Claude CLI hoje; ela usa providers de imagem salvos em Configurações.

Diagnóstico curto

  • MiniMax aparecia porque era o provider padrão de imagem salvo. API key não troca provider/modelo sozinha.
  • Codex/Gemini CLI geram texto no Open Studio, mas ainda não são um provider de imagem no app.
  • Agora existe POST /api/generate/thumbnails para gerar lote real de 1 a 10 thumbnails.
  • Agora existe fallback: se o provider padrão exige chave e está sem chave, o backend tenta Pollinations e devolve diagnóstico de fallback.

Teste externo feito

  • https://gen.pollinations.ai/image/... sem chave: retornou 401 Unauthorized.
  • https://image.pollinations.ai/prompt/... sem chave: retornou 200 OK com image/jpeg.
  • Por isso o adapter usa endpoint legado quando não há chave e endpoint novo quando há chave.

Smoke local feito

  • /api/generate/thumbnails retornou 200 e salvou imagem real em /generated/thumbnails/....
  • Teste com provider pollinations sem chave também retornou 200.
  • npm run typecheck passou depois das mudanças.

O caminho mais fácil para testar

Para testar fluxo real sem pagar OpenAI agora, use Pollinations como provider de imagem. Se o endpoint legado continuar aceitando sem chave, funciona direto. Se ele bloquear, o caminho mais rápido é pegar uma chave gratuita/publishable em enter.pollinations.ai e salvar no provider Pollinations.

Para qualidade profissional estável, os caminhos continuam sendo pagos ou com quota: OpenAI GPT Image, Google Gemini/Imagen, fal.ai, Replicate, Together/FLUX, BFL.

Regras implementadas

  • A interface segue o idioma selecionado: PT-BR, ES-ES ou EN-US.
  • Novas gerações de texto seguem o idioma selecionado.
  • Prompts técnicos de imagem são enviados em inglês para melhorar resultado.
  • Qualquer texto visível na imagem deve ficar no idioma selecionado.
  • Thumbnails agora podem ser geradas em lote de até 10 imagens reais.

Fontes verificadas

  • Pollinations API docs: base gen.pollinations.ai, autenticação e endpoints.
  • Google Gemini image generation docs: Gemini/Nano Banana e Imagen via API.
  • OpenAI image generation docs: Image API, Responses API e modelos GPT Image.
