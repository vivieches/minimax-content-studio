# Open Studio OpenDesign Buildout - Sprints And Specs

Data: 2026-05-11

## Implementation Status

- Done: Sprint 0 storage/test isolation, Sprint 1 daemon foundation, Sprint 2 project store, Sprint 3 run lifecycle, Sprint 4 agent runtime loop, Sprint 5 fallback chain and diagnostics, Sprint 6 provider/model discovery, Sprint 7 full media tool, Sprint 8 research and CTR/SEO titles, Sprint 9 captions, Sprint 10 prompt composer, skills, memory and brand kit, Sprint 11 package critique, Sprint 12 file workspace and live artifacts.

## Sprint 0 - Stabilize Current Main

Goal: keep the current app usable while the daemon buildout starts.

Deliverables:

- Confirm current tests, lint and build.
- Fix stale settings/default provider persistence if still reproducible.
- Make title pack generated by pipeline visible on `/content`.
- Ensure image route uses saved image default and never silently falls back to MiniMax.
- Add clear error when Codex CLI fails due model/account/version and no BYOK fallback exists.

Tests:

- `rtk npm test`
- `rtk npm run lint`
- `rtk npm run build`
- Browser smoke for settings, pipeline, content, thumbnails.

## Sprint 1 - Real Daemon Foundation

Goal: introduce a separate long-running daemon without breaking Next UI.

Specs:

- Add daemon entrypoint.
- Add daemon HTTP server.
- Add daemon config, health, logs and status.
- Add daemon client for Next routes.
- Add startup script that can run web + daemon locally.
- Keep Next API routes as proxy/compat layer.

Acceptance:

- `GET /api/health` reports Next and daemon health.
- Daemon writes `.open-studio/daemon/daemon.log`.
- Next routes can forward to daemon when daemon is running.
- If daemon is down, UI shows actionable local runtime error.

Specs:

- Add project model.
- Add safe project path validation.
- Add file tree under `.open-studio/projects/<projectId>/files`.
- Add project manifest and package file.
- Add project routes.
- Migrate existing asset/export creation to optionally attach to project.

Acceptance:

- User can create/list/open project.
- Generated script writes to project file.
- Generated image writes to project file and global asset reference.
- Project export includes files.

## Sprint 3 - Run Lifecycle And Events

Goal: turn generation into observable runs.

Specs:

- Add run registry.
- Add run events JSONL.
- Add SSE event stream.
- Add cancel/retry/resume contract.
- Normalize statuses.
- Show run status in UI.

Acceptance:

- Starting pipeline creates a run.
- Run status transitions from queued to running to succeeded/failed/cancelled.
- Errors persist with raw category and redacted details.
- UI can reload and still see last run status.

## Sprint 4 - Agent Runtime Loop

Goal: replace prompt-in/text-out with evented agent runs.

Specs:

- Port runtime registry shape from OpenDesign.
- Add parser adapters for Claude, Codex, Gemini, OpenCode and plain stream first.
- Add Copilot, Qoder, Pi, ACP and DeepSeek adapters next.
- Add capability gating.
- Add prompt transport via stdin/temp file/argv guard.
- Add per-agent env and permission defaults.

Acceptance:

- Claude, Codex, Gemini and OpenCode emit normalized events.
- Codex plugin/skill noise does not hide model/account errors.
- UI displays detected vs connected correctly.
- Agent cancellation stops process.

## Sprint 5 - Fallback Chain And Diagnostics

Goal: make failures understandable and recoverable.

Specs:

- Add fallback policy engine.
- Add fallback run events.
- Add provider/agent diagnostics.
- Add Claude diagnostics.
- Add connection test categories across agent/provider/media/research.

Acceptance:

- If selected CLI fails, UI shows why and what fallback ran.
- If no fallback exists, UI tells which setting is missing.
- Connection tests classify auth, rate, model, timeout, unavailable, invalid URL and spawn failures.

## Sprint 6 - Provider Catalog And Model Discovery

Goal: make provider/model data single source of truth.

Specs:

- Consolidate text/image/media provider manifests.
- Add hidden media providers.
- Add live model discovery with cache where possible.
- Add unsupported/integrated labels.
- Add model capability metadata.

Acceptance:

- Settings provider/model selectors never show stale models.
- API key presence does not change defaults automatically.
- Defaults decide generation route.
- Unsupported providers are visible only where explicitly useful.
